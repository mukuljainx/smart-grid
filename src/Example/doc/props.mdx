## schema

<TagGroup>
  <Tag color="red">Required</Tag>
  <Tag color="blue">Array</Tag>
</TagGroup>
<br />
Schema should be collection of object of `ISchema` type
<br />

1. **width**: Should be in pixels.
2. **get**: Each cell does not need all the data from row using getter one pick limited keys from the row, as each cell template is rendered inside Pure Component it will be only updated when those values are changed.
3. **template**: React Component, component will receive props from getter function also `rowIndex` as props.
4. **pinned**: "LEFT" or undefined, will pin following coloumn to left.
5. **header**: React Component

### loadMore

<TagGroup>
  <Tag color="red">Optional</Tag>
  <Tag color="blue">Function</Tag>
</TagGroup>
<br />

Function of type `() => void` should be provided which will be invoked when user is about reach the end of grid. It will be invoked once and will be not rendered again till the data is updated.

<Divider />

### buffer

<TagGroup>
  <Tag color="red">Optional</Tag>
  <Tag color="blue">Number</Tag>
</TagGroup>
<br />

**default**: 5

Extra rows to be rendered below and above the visible table for smooth user experiance, for e.g. if row 15 to 25 are visible to user and buffer is 5, then rows from 10 to 30 will be rendered.

### dynamicRowHeight

<TagGroup>
  <Tag color="red">Optional</Tag>
  <Tag color="blue">Boolean</Tag>
</TagGroup>
<br />

**default**: false

When `true` each row height will be calculated dynamically and given row height will be considered as minimun row height.
The performace will not be equivalent fixed height row. To **match the performace** it advised to provided paginated data via usig `loadMore` prop, as once the height and position of rows are calculated they are cached and re-render for same rows are 200% faster.

If any cell data updates which can change the height call, `gridActions.refreshRows([...indexs])` it will render the changed rows with new height.
